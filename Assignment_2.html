[Info1] Assignment 2: Working with Real Data
(35 Points)

The goal of this assignment is to work with real life data. You will practice the basics of data manipulation and analysis, as well as data visualization.

Deadline

The deadline for this assignment is on January 15, 2025 at 23:59:00. This is a hard deadline; therefore, no late submissions will be accepted.

You submit your assignment in the TeachCenter. Multiple submissions are allowed, so you can submit your code as often as you want. The last submission before the deadline will be graded; we have no access to previous versions.

Group Work

This assignment is group work - that means it must be completed in groups of 3 students (i.e., you cannot do the assignment alone).
Please use the group selection tool in the "Assignment 2" section in the TeachCenter to register your group. The forum "Group Search" in the TeachCenter can be used to find partners for this assignment. If you cannot find a group partner, you will be randomly assigned to another group/person.

Submission Files

When you have completed the assignment, you will submit the following files:

  • assignment2.ipynb: This file contains all the code that you write for Assignment 2.
  • assignment2_report.pdf: This file contains your report.
  • All data files that you used: Please also submit all data files that you downloaded and used in your code. These data files have to be the original files that you downloaded from https://data.oecd.org/ (file name and content have to be identical).

Do not submit any other files. Make sure you use Python version ≥3.10.

Only one person from your group has to submit the files in the TeachCenter.

Important: Make sure to clear all cell output in your assignment2.ipynb before you submit it!

Style Guide for Code Quality

Follow the rules and guidelines outlined in this document.

Severe style violations will lead to the subtraction of up to 7.5 points.

In this assignment, you will design your code yourself. This means that we do not tell you which functions you should write and in what way. Instead, after reading the requirements for the assignment, you should plan the design of your code. Think about:

  • Which functions do you need? What should they do?
  • Which parameters and return values should your functions have?

Make sure that you write no code outside of functions!
The only code outside of functions should be the function call itself and the import statements.

Allowed Packages

You are allowed to use everything in the Python Standard Library, as well as the following libraries

pandas
matplotlib
plotly
numpy
pycountry
pycountry_convert

The packages pycountry and the pycountry_convert can be used to convert ISO alpha-3 country codes to their corresponding full country name (e.g., AUT -> Austria) and to retrieve the corresponding continent using a country code (e.g., AT -> Europe).

Assignment Structure

This assignment consists of three main parts:

1. Python Code: You will write code to analyze and visualize real world data. You will design the structure of the code yourself (e.g., which functions, which parameters, which return values, etc.).
2. Report: You will summarize your code design as well as the results obtained from your data analysis and visualization in a written report.
3. Presentation: During your presentation, you will present your code design and the results of your data analysis & visualization.

Please refer to the sections below for more details on the individual parts of the assignment. All three parts of the assignment are mandatory; not completing one of the parts will lead to zero points on the assignment.

Choose Your Own Data

In this assignment, you will be able to choose yourself which data you want to analyze. You can choose from all data files provided by https://data.oecd.org/.

How to download data for the assignment:

  1. Go to https://www.oecd.org/en/data.html

  2. Search the data by keywords or click on "Datasets" :

  3. You are now shown all datasets. You can filter by categories on the left. Take your time to browse through the different categories to see what interests you. You will need several different datasets for this assignment.

  4. For example, you could have decided that you are interested in Alcohol consumption database. The website shows you an overview of the data:

  5. To download the data, click on "Download", and then on "Unfiltered data in tabular text (.csv)" to download the full dataset. Use this downloaded file as the starting point for your analysis. Do not rename the downloaded .csv files or change the content of the files manually (e.g., do not open them in Excel and manually select or delete data). Instead, always use pandas to select the data that you need for your plots.

    In case your chosen dataset has missing values, discuss with your group partner(s) on how to handle the missing data.

Important: Please make sure that the total size of all your chosen datasets is below 95MB.

Tasks

Part 1: Data Analysis & Visualization Code - Required Plots

In the assignment, you will visualize your chosen data using line plots, bar charts, and scatter plots. Two different plots of each type (i.e., visualizing different data) are required for the assignment. Make sure that you design your functions well.

Requirements for all plots

Each plot that you design should have, as a minimum:

  • A title
  • Axis labels with used units if applicable.
  • Axis tick labels
  • An appropriate scale for the axes

Where appropriate, also include a legend (for example, to distinguish between the lines in a line plot with multiple lines).

Line plots: Visualize data over time (4P)

Create at least two line plots that show data over time. Each line plot should use a different data file. Each line plot should compare a variable in different countries over time (i.e., each line plot should have more than one line).

Examples for what your line plots could show:

  • A comparison of the life expectancy in five countries over time, using the dataset https://data.oecd.org/healthstat/life-expectancy-at-birth.htm.
  • The percentage of the working age population over time, in different countries (https://data.oecd.org/pop/working-age-population.htm)
  • The fertility rate in different countries over time (https://data.oecd.org/pop/fertility-rates.htm)
  • The quarterly employment rate in different countries over time (https://data.oecd.org/emp/employment-rate.htm)
  • etc.

Bar charts: Compare data from different countries (4P)

Create at least two bar charts that compare data from different countries. Each bar chart should use a different data file.

Examples for what your bar charts could show:

  • The percentage of women who agree that a husband/partner is justified in beating his wife/partner under certain circumstances, in different countries (https://data.oecd.org/inequality/violence-against-women.htm)
  • The poverty rate in different countries for a specific year (https://data.oecd.org/inequality/poverty-rate.htm#indicator-chart)
  • The total number of magnetic resonance imaging (MRI) exams performed in different countries in a specific year (https://data.oecd.org/healthcare/magnetic-resonance-imaging-mri-exams.htm)
  • etc.

Scatter plots: Visualize the relationship between different variables (7P)

Create at least two scatter plots, each showing the relationship between two variables.
For each scatter plot, choose two different datasets from https://data.oecd.org/ that contain variables that you are interested in and which you think might have an interesting relationship. To further analyze the relationship between two variables, also calculate the correlation coefficient between the variables. For calculating the correlation coefficient, you can use, e.g., pandas.Series.corr().

Examples for what your scatterplots could show:

  • Child vaccination rates (https://data.oecd.org/healthcare/child-vaccination-rates.htm) compared to GDP per capita (https://data.oecd.org/gdp/gross-domestic-product-gdp.htm)
  • GDP per capita (https://data.oecd.org/gdp/gross-domestic-product-gdp.htm) compared to life expectancy (https://data.oecd.org/healthstat/life-expectancy-at-birth.htm)
  • Doctors' consultations per capita (https://data.oecd.org/healthcare/doctors-consultations.htm) compared to life expectancy (https://data.oecd.org/healthstat/life-expectancy-at-birth.htm)
  • Inflation measured by consumer price index (CPI) (https://data.oecd.org/price/inflation-cpi.htm) compared to short-term interest rates (https://data.oecd.org/interest/short-term-interest-rates.htm)
  • etc.

Part 2: Report (12P)

When you have finished your data analysis and visualization, write a report (in .pdf format) that summarizes your code design and your results.

Your report must include, at a minimum, the following sections:

  • A title page containing the names and matriculation numbers of all authors, as well as your group number.
  • Introduction: In this (short) section, you introduce your work and give the reader some general information such as which dataset you used and why you used them.
  • Code Design: This section contains a description of how you structured your code. In addition to a description of the general structure of your code, also answer the following questions:
    • Why did you design the functions the way you did?
    • In what manner do you call your functions (refer to your function by their names)?
      Do not include any actual code in this section!
  • Data Visualization and Analysis: This section contains every plot that you created. Each plot should have a corresponding caption and a description that answers the following questions:
    • Which data did you use in the plot? Why did you use this data for comparison?
    • What can you see in the plot? Interpret your results.
      For the scatter plots, include your interpretation of the correlation coefficients in the description:
      • Does the data correlate?
      • Did you get the correlation that you expected?
    • Why is this plot type suitable for your chosen data?

Important: Double check that you included all plots in your report! You will not receive any points for plots that you did not include in your report. This also means that if you don't submit a report at all, you will get zero points on the assignment.

Keep your report short and concise, it should not be more than 8 pages (including the plots, excluding the title page). Submit your report as a .pdf file.
Make sure that your report includes your names, group number, and matriculation numbers.

Part 3: Presentation (8P)

Finally, prepare a presentation about your project where you present your code design and your results. The presentations will take place in January (dates will be announced) and should be 10 minutes long (make sure that it’s not longer than 10 minutes!). Each group needs to present their projects and all students in your group should do a part of the presentation.

During the presentation, make sure that you present:

  1. The design and functionality of your code
  2. The results of your data analysis and visualization

Make sure to prepare slides for your presentation.

Also be prepared to demonstrate parts of your code on request and answer detailed questions about your code and design. All students in your group have to understand all parts of the code.
